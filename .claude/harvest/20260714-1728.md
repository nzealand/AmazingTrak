## GOTCHAS

- `L.divIcon` does not auto-size to its HTML content. For a label marker whose width varies with content (e.g. a train-number badge that can be 1-4 digits), setting `iconSize` to the label's pixel dimensions doesn't work — the correct pattern is `iconSize: [0,0]`, `iconAnchor: [0,0]`, and doing the actual centering inside the divIcon's HTML via `position: absolute; transform: translate(-50%, -50%)` on the inner div. Fixed-pixel `iconAnchor` guesses don't track content width. See `templates/map.html`.
- `GET /api/live-trains` returned 404 in a fresh local run (fresh seeded DB, no real Amtraker data flowing). Not yet root-caused whether this is because live trains are gated by an admin setting (`LIVE_TRAINS_ENABLED`, checked client-side in `templates/map.html`) that's off by default on a fresh DB, or something else server-side in `livetrains.go`. Worth confirming before assuming the endpoint is broken next time it's needed for local testing.

## ARCHITECTURE

- Live train markers on `/map` are rendered entirely client-side: `templates/map.html` fetches `/api/live-trains` (JSON with `trainNum`, `lat`, `lon`, `status`, `delayMin`, `speed`, `heading`, `nextStation`, `trainSlug`, `displayName` per train — see `livetrains.go:30,49`), then builds Leaflet markers in JS with a `STATUS_COLOR` map (`ontime`/`late`/`verylate`) driving marker fill color and popup text. No server-side template rendering of individual train markers — the Go side only decides whether the feature is enabled and serves the JSON.
- `templates/map.html` polls `/api/live-trains` every 90s (`setInterval(refreshTrains, 90000)`), matching the upstream Amtraker refresh cadence referenced in on-page attribution text.
